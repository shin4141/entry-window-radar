# Industry Slope Timeline v0.1

## Purpose

Industry Slope Timeline shows the current As-of position of a project, workflow, or operator inside a reference industry lifecycle.

It is a display-stage context figure, not a prediction chart. It helps the user and future Codex/AI understand what reference market or workflow line is being used, where the operator sits now, and what posture is appropriate before the next gate.

## Relationship To Quest Position Map

Quest Position Map compares candidate quests by proof distance, habitat/reward potential, enemy pressure, and survival/relationship risk.

Industry Slope Timeline adds the reference-market time layer behind that map:

- what past context shaped the opportunity
- where the workflow appears now
- what near-future conditions should be rechecked
- why the recommended posture is PROOF, INCUBATE, SHORT CYCLE, HOLD, or AVOID

The timeline does not replace Quest Position Map. It provides a companion Figure 2 for Quest Snapshot / TimeTube Archive use.

## Relationship To Reference Chart Quality

The timeline must name the reference line being used.

For example, "AI market overall" is usually too broad for a workflow-compression quest. A better reference line is closer to the actual habitat, such as:

- human manual work
- recurring workflow cost
- AI compression potential
- payment / switching pressure
- output equivalence proof

If the reference chart is broad, weak, unknown, or borrowed from an adjacent market, confidence should stay lower and the chart must not imply launch permission.

## Axis Definitions

### X-Axis: Time

- past
- now
- near future / horizon

The left side shows prior conditions. The center shows current As-of position. The right side is a recheck zone, not a guaranteed future.

### Y-Axis: Industry Slope / Adoption Momentum / Opportunity Pressure

- low
- rising
- steepening
- crowded / saturated / absorbed

Higher position means the reference line appears to have more adoption pressure, urgency, or competitive movement. It does not mean guaranteed success.

## Timeline Phases

Use simple phase labels:

- pre-entry
- early adoption
- steepening
- crowded
- post-FOMO
- declining
- unclear

These are qualitative display phases, not scored market predictions.

## Operator Position Marker

The chart should mark the current operator/project position clearly.

The marker should name:

- current posture: PROOF / INCUBATE / SHORT CYCLE / HOLD / AVOID
- operator position, such as PROOF SEEKER, builder, early adopter, investor, late entrant, or avoid
- current reason for that posture

## Recheck Condition

Every timeline should include a bounded recheck condition.

Examples:

- paid repeat appears
- output equivalence is proven
- competition increases
- LLM or SaaS absorption appears
- reference chart quality improves or weakens
- switching pressure changes

## What This Chart Answers

This chart answers:

```text
Where are we in the industry lifecycle, and what posture should the operator take now?
```

It can also help answer:

- Which reference line are we actually using?
- Is the operator too early, on time, or late?
- Is the right action PROOF, INCUBATE, SHORT CYCLE, HOLD, or AVOID?
- What must be rechecked later?

## What This Chart Does Not Answer

This chart does not answer:

- the exact future market size
- guaranteed growth
- investment timing
- automatic launch permission
- whether Codex may implement beyond the current Gate
- whether the project will succeed

## Governance Boundaries

Industry Slope Timeline is an As-of context chart.

It must not become:

- automatic market analysis
- market research
- external API use
- web research
- forecasting logic
- scoring automation
- launch-order automation
- full Quest Map implementation
- Quest Snapshot runtime output
- PDF generation
- screenshot automation
- external posting material

## Completion Line

Industry Slope Timeline v0.1 defines a static As-of companion figure for showing where an operator/project sits inside an industry or reference-market lifecycle without becoming prediction, market sizing, scoring automation, or launch permission.
