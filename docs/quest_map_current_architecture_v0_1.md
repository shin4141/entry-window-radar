# Quest Map Current Architecture v0.1

## Purpose

Quest Map Current Architecture v0.1 summarizes what Quest Map / Quest Snapshot currently is, what each artifact does, what is already PASS, what remains HOLD, and what should happen next.

It exists so future Codex/AI sessions can restart from repo files without confusing first-use charts, evidence notes, saved snapshots, and recurring TimeTube trajectory.

## Core Architecture

Quest Map currently has five conceptual layers.

### Initial-Position Layer

- Quest Position Map
- Industry Slope Timeline

This layer helps place a quest as-of now. It is useful for first positioning, but it is not the main recurring value by itself.

### Evidence / Explanation Layer

- Codex Interpretation Note / Evidence Basis

This layer explains why visuals are placed or classified as they are. It carries the reasoning and missing evidence that a future Codex/AI should read before changing scope.

### Archive / Handoff Layer

- Quest Snapshot / TimeTube Archive

This layer saves As-of decision artifacts so future sessions can know what they now own, why the previous decision was made, and what must not be broadened.

### Recurring Value Layer

- Snapshot Trajectory / Drift Delta

This layer compares saved snapshots over time. It is where recurring value begins: compounding, stagnation, drift, risk escalation, and readiness to advance become visible across snapshots.

### Future Risk Layer

- Survival / Relationship Risk Panel

This layer is currently HOLD. It may later support decisions involving survival exposure, employees, family, payroll/runway, relationship value, regret guard, and counterparty viability.

## Artifact Roles

### A. Quest Position Map

Role:
Shows which quest the operator should consider now.

Answers:

- Which quest is close or far?
- Which quest has durable habitat / reward potential?
- Which quest is PROOF / INCUBATE / HOLD / SALVAGE / AVOID?

Status:
PASS enough as visual prototype.

Limit:
Initial-position chart. Not the main recurring value by itself.

### B. Industry Slope Timeline

Role:
Shows where the selected reference niche sits in its lifecycle.

Answers:

- Is this niche early, steepening, crowded, absorption-risk, mature, or salvage?
- Is the operator late or still in a valid proof window?

Status:
PASS enough as visual prototype.

Limit:
Not a prediction chart. Not the whole AI market. It shows the selected niche's entry-window lifecycle.

### C. Codex Interpretation Note / Evidence Basis

Role:
Explains why the visuals are placed or classified as they are.

Carries:

- source-backed / observed / estimated / unknown / assumption labels
- commoditization and absorption risk
- counterarguments
- missing evidence
- do-not-do boundary
- recheck condition

Status:
PASS as spec.

### D. Quest Snapshot / TimeTube Archive

Role:
Saves Quest Map outputs as As-of artifacts.

Purpose:
Future Codex/AI can know:

- what it now owns
- current Gate
- next action
- do-not-do boundary
- completion line
- why the previous decision was made

Status:
PASS as spec.

### E. Snapshot Trajectory / Drift Delta

Role:
Compares multiple Quest Snapshots over time.

Recurring value:
Shows whether the project is:

- COMPOUNDING
- STALLED
- DRIFTING
- RISK ESCALATING
- READY TO ADVANCE
- SHOULD HOLD
- SHOULD CAP
- SHOULD BLOCK

Status:
PASS as spec and visual prototype.

### F. Quest Snapshot Visual Design System

Role:
Defines visual rules so future artifacts are readable, Japanese-friendly, screenshot-ready, and useful for Codex handoff.

Status:
PASS as spec.

### G. Survival / Relationship Risk Panel

Role:
Future optional panel for decisions involving:

- company survival
- employees
- family
- payroll / runway
- relationship value
- regret guard
- counterparty viability

Status:
HOLD.

Do not implement or design now.

## Current Product Interpretation

Quest Map is not just a market analysis tool.

It is:

- a quest-selection map
- an As-of decision artifact
- a Codex handoff object
- a TimeTube trajectory system
- a drift-prevention layer for AI-built apps / OSS / projects

## What Creates Recurring Value

Initial charts are useful for first positioning.

Recurring value comes from:

- saved Quest Snapshots
- Codex reading prior Gate / next action / do-not-do boundary
- Snapshot Trajectory comparing changes over time
- detecting compounding, stagnation, drift, or risk escalation
- reducing repeated explanation
- preventing uncontrolled AI/Codex expansion

## Current Non-Goals

Quest Map is not:

- prediction
- investment advice
- automatic launch permission
- VC scoring
- market oracle
- external research automation
- full project management tool
- replacement for human Seat
