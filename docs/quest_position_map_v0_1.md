# Quest Position Map v0.1

## Purpose

Quest Position Map is a screenshot-ready visual prototype for placing quest candidates by proof distance, habitat/reward potential, enemy pressure, and survival/relationship risk.

It helps the operator compare which quest may be close, far, durable, short-cycle, enemy-heavy, or unsuitable for action under the current Gate.

Quest Position Map answers:

```text
Which quest should the operator consider now?
```

It is a display-stage prototype, not a full Quest Map implementation.

## Axes Definition

### X-Axis: Proof / Reward Distance

Left means close.

Right means far.

This axis asks how far the operator is from first meaningful proof, reward, paid signal, case study, strategic artifact, or recheckable evidence.

### Y-Axis: Habitat / Reward Potential

Bottom means low, one-off, weak habitat, or limited option value.

Top means durable, high option value, repeatable habitat, or strong future reuse potential.

This axis asks whether the quest can become more than a one-time proof.

## Point Definition

Each point represents one quest candidate.

The prototype uses hand-authored illustrative sample data only.

The point position is a display-stage placement, not a precise probability, forecast, market score, or automated recommendation.

## Label / Action Definition

Each point may show one recommended action:

- LAUNCH
- PROOF
- SHORT CYCLE
- INCUBATE
- HOLD
- AVOID
- SALVAGE

The action is an As-of posture for the sample quest, not launch permission.

## Enemy / Competition Pressure Representation

Enemy / Competition Pressure is represented by a ring or outline around the point.

A larger or darker ring means higher pressure from incumbents, substitutes, major players, capital requirements, execution depth, or fast-moving competitors.

The ring is a visual cue, not a scoring engine.

## Survival / Relationship Risk Representation

Survival / Relationship Risk is represented by a warning marker or note.

This risk may include cashflow exposure, payroll/runway pressure, relationship damage, local reputation constraints, dependency asymmetry, shared sink risk, or Regret Guard concerns.

The warning marker does not decide for the human.

It tells Codex/AI and the operator to keep the risk visible before broadening scope.

## What This Chart Answers

Quest Position Map helps answer:

- which quest is closest to proof
- which quest may have durable habitat
- which quest is likely short-cycle or salvage-only
- which quest is too enemy-heavy for the current operator position
- which quest should remain HOLD or AVOID
- what the one next action is
- what Codex/AI must not broaden

## What This Chart Does Not Answer

Quest Position Map does not answer:

- whether the market will definitely grow
- whether success is guaranteed
- whether the human should surrender the Seat
- whether Codex may implement beyond the current Gate
- whether the sample data is real market research
- whether the displayed position is a probability
- whether the action is automated launch permission

## Relationship To Quest Snapshot / TimeTube Archive

Quest Position Map can become one optional visual attachment inside a future Quest Snapshot.

The snapshot may save:

- the current Gate
- recommended action
- one-line judgment
- one next action
- do-not-do boundary
- recheck condition
- Seat owner
- Regret Guard note when relevant
- Quest Position Map image when explicitly gated

This lets future Codex/AI sessions reopen the As-of position without asking the user to restate the project state.

This phase does not implement snapshot generation.
