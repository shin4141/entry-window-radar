# Quest Position Map v0.1 Example

## Purpose

This example explains the hand-authored sample data used in `outputs/quest_position_map.svg`.

The SVG is only a visual prototype.

It does not use real market research, automatic scoring, or Quest Map recommendation logic.

## Why This Is Only A Prototype

This example does not implement:

- full Quest Map scoring
- automatic recommendation logic
- market research
- external APIs
- Quest Snapshot runtime output
- PDF generation
- screenshot automation
- CLI behavior changes

The positions are display-stage placements chosen to test whether the chart communicates quest posture at a glance.

## Future Snapshot Use

A future Quest Snapshot may save this chart as an As-of artifact alongside:

- current Gate
- selected quest
- one-line judgment
- one next action
- do-not-do boundary
- recheck condition
- Seat owner
- Regret Guard note when relevant

That would let a future Codex/AI session understand what it now owns before continuing.

Snapshot saving is not implemented in this phase.

## Completion Line

The Quest Position Map v0.1 example explains the fixed sample data and placement rationale for the static SVG prototype without adding scoring, rendering automation, snapshot output, or external scope.
