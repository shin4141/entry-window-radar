# Contributing

Entry Window Radar is currently prompt-first and template-first.

Runtime implementation is HOLD. `outputs/quest_snapshot.md` generation is HOLD.

Good contributions include:

- README clarity
- examples
- prompt improvements
- UNKNOWN handling improvements
- documentation fixes
- public-safe dogfood reports
- clearer boundary language for Codex/AI handoff

Not accepted yet:

- runtime engine
- hooks, MCP, pluginization, or execution engine work
- scoring or drift scoring
- PDF, PNG, or screenshot automation
- snapshot comparison
- external API or market research automation
- broad product redesign

The human keeps the Seat. Contributions should preserve the Current Gate, one next action, Do-Not-Do Boundary, Recheck Condition, and Completion Line.

Before opening a pull request, read `README.md`, `STATUS.md`, and `AGENTS.md`.
